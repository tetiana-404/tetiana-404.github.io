Cloud ETL Project

ETL Process in AWS

This project demonstrates my ability to build an ETL (Extract-Transform-Load) process in the Amazon Web Services (AWS) environment using S3, AWS Lambda, and DynamoDB.

Project Brief

The goal was to process data from a JSON file containing information about the 100 best books in human history. The project involved cleaning, transforming, and loading the data into a DynamoDB table.

ETL Steps
  • E - Extract: Loaded JSON files from AWS S3 bucket into AWS Lambda function.
  • T - Transform: Performed data cleaning, reordering columns, and language code conversion using Pandas inside Lambda.
  • L - Load: Loaded the transformed data into AWS DynamoDB table.
AWS Services Used
  • Amazon S3 (for data storage)
  • AWS Lambda (for data processing)
  • Amazon DynamoDB (for data storage)
  • IAM Roles (for permissions)
Python Libraries Used
  • Pandas
  • JSON
  • Boto3 (AWS SDK for Python)
Dataset Overview

Original file: 100_best_books.json

Dataset contained the following fields:

Column Name Type Description
authorStringBook author
countryStringAuthor's country
imageLinkStringLink to cover image
languageStringLanguage of the book
linkStringWikipedia link
pagesIntegerNumber of pages
titleStringBook title
yearIntegerYear of publication
Transformation Details
  • Dropped unnecessary columns (e.g., country, link, imageLink)
  • Reordered columns to match expected output: author, title, year, language, pages
  • Converted language names to ISO 639-1 language codes using language.json
  • Optional: Year formatting (e.g., "1700 BC", "400 AD")
Deployment Results
  • Transformed JSON data was successfully uploaded into DynamoDB
  • Each book is stored as an individual item in DynamoDB
  • Lambda role was configured with permissions to read from S3 and write to DynamoDB
Deliverables
  • Python script: scratch_of_solution.py
